# Story OS — 产品需求文档 (PRD)

> 版本: 0.1.0-MVP | 日期: 2026-06-26 | 状态: Draft

---

## 1. 产品愿景

Story OS 是一个面向长篇小说作者、剧本作者、跑团主持（GM）、世界观创作者的 AI 写作工具。

**Story OS 不是一个通用的 AI 写作工具。** 它的核心价值在于长期故事记忆、分支管理、一致性检查和作者控制权。它不是简单的"输入一句话→AI 续写一段"工具，而是一个**能管理长篇故事、人物、时间线、分支、伏笔和世界观一致性的系统**。

核心洞见：现有 AI 写作工具在生成 500 字时表现不错，但在管理 50 万字的复杂叙事时会严重退化——人物性格漂移、伏笔丢失、世界观矛盾、时间线混乱。Story OS 通过多层记忆架构和结构化故事图谱解决这个问题。

---

## 2. 目标用户

| 用户类型 | 核心场景 | 痛点 |
|---------|---------|------|
| 长篇小说作者 | 维护 10 万+ 字的故事，需要 AI 辅助续写 | AI 记不住前文设定，生成内容前后矛盾 |
| 剧本/影视编剧 | 多线叙事、场景管理、对白润色 | 人物多了之后 AI 混淆角色关系 |
| 跑团主持 (GM) | 管理世界设定、NPC、事件时间线 | 需要快速检索世界细节，保持一致性 |
| 世界观创作者 | 构建架空世界的设定集 | 大量互相关联的设定条目，需要结构化存储 |

## 3. 核心概念

### 3.1 故事图谱 (Story Graph)

故事不是单一线性文档，而是一个**有向图**：

```
[Chapter 1] → [Chapter 2] → [Chapter 3 - Branch A] → [Ending A]
                           ↘ [Chapter 3 - Branch B] → [Ending B]
```

- **Scene（场景）**：最小叙事单元，包含一个时间/地点内的完整事件
- **Chapter（章节）**：多个 Scene 的逻辑组合
- **Branch（分支）**：从某个节点分叉出去的平行叙事线
- **Story Arc（故事弧）**：跨多个节点的主题或情节线

### 3.2 双状态系统

所有 AI 生成的内容必须先进入 **Draft（草稿）** 状态，用户审核确认后才进入 **Canon（正史）**：

- **Draft**：AI 生成的暂存内容，可编辑、可废弃、不影响故事宇宙
- **Canon**：用户确认后的"正式设定"，AI 后续生成将以 Canon 为准

### 3.3 五层记忆架构

详见 [AI_MEMORY.md](./AI_MEMORY.md)，概要：

| 层 | 名称 | 职责 | 存储方式 |
|---|------|------|---------|
| L0 | 原文层 | 完整原文存储 | Postgres TEXT |
| L1 | 摘要层 | 节点级短/长摘要，递归向上聚合 | Postgres TEXT |
| L2 | 向量检索层 | 语义相似检索 | pgvector (1536d) |
| L3 | 故事圣经层 | 结构化世界观事实 | Postgres JSONB |
| L4 | 事件账本层 | 不可变事件时间线 | Postgres TABLE |

## 4. MVP 功能范围

### 4.1 故事管理

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F1 | 创建/删除故事项目 | P0 | 一个用户可有多个故事 |
| F2 | 创建/编辑/删除 Scene | P0 | 文本编辑（textarea），支持 Markdown |
| F3 | Scene 排序与拖拽重排 | P1 | MVP 用手动序号，不做拖拽 UI |
| F4 | 故事分支（分叉） | P2 | MVP 先做线性故事，分支数据结构预留 |

### 4.2 AI 写作

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F5 | AI 续写 | P0 | 基于当前节点及上下文，生成后续内容 |
| F6 | AI 扩写 | P0 | 选中一段文字，让 AI 扩充细节 |
| F7 | AI 润色 | P0 | 语法、文笔优化，保持原有风格 |
| F8 | AI 压缩 | P1 | 精简文字，保留核心信息 |
| F9 | AI 改写风格 | P1 | 切换叙事风格（如第一人称↔第三人称） |
| F10 | AI 矛盾检查 | P1 | 检查当前内容与已有设定的矛盾 |

### 4.3 故事宇宙管理

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F11 | 人物管理（CRUD + 描述） | P0 | 每个故事维护人物列表 |
| F12 | 地点管理（CRUD + 描述） | P1 | 世界观地点 |
| F13 | 世界观事实管理 | P1 | 键值对形式的世界观规则 |
| F14 | 伏笔追踪 | P2 | 标记和检索伏笔 |
| F15 | 事件时间线 | P2 | 不可变事件账本 |

### 4.4 Draft/Canon 工作流

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F16 | AI 输出标记为 Draft | P0 | 所有 AI 生成内容默认带 draft 标记 |
| F17 | 用户审核确认（Promote to Canon） | P0 | 确认后内容成为正史 |
| F18 | 用户编辑 Draft | P0 | 确认前可修改 AI 生成内容 |
| F19 | 废弃 Draft | P0 | 删除不需要的 AI 生成内容 |

### 4.5 导入/导出

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F20 | 粘贴/输入文字 | P0 | 直接在编辑器中输入 |
| F21 | 上传 .txt/.md 文件 | P1 | 导入现有文稿 |
| F22 | 导出为 Markdown | P1 | 导出完整故事 |

## 5. 明确不做（MVP Out of Scope）

- ❌ 用户注册/登录系统（MVP 使用单用户模式，本地或硬编码用户 ID）
- ❌ 多人协作/评论/分享
- ❌ 社区功能
- ❌ 支付/订阅
- ❌ 移动端 App
- ❌ 富文本编辑器（MVP 用 textarea + Markdown 渲染预览）
- ❌ 版本历史/Git 式 diff
- ❌ 多语言支持
- ❌ 第三方 API 集成（除 OpenAI）
- ❌ 离线支持/PWA
- ❌ 故事分支图谱可视化（数据结构预留，UI 不实现）

## 6. 用户旅程（MVP 核心流程）

### 6.1 创建故事并续写

```
1. 用户打开 App → 看到故事列表（空状态）
2. 点击"创建新故事" → 输入故事标题、一句话简介
3. 进入故事详情页 → 看到空的 Scene 列表
4. 点击"新建 Scene" → 进入编辑器
5. 在编辑器中输入/粘贴已有文字（或直接让 AI 生成开头）
6. 点击"AI 续写" → 系统收集上下文 → 调用 OpenAI → 生成内容进入 Draft 区
7. 用户查看 Draft → 可编辑 → 点击"确认" → Draft 转为 Canon Scene
8. 重复步骤 6-7，逐步推进故事
```

### 6.2 管理人物

```
1. 在故事详情页 → 切换到"人物"标签
2. 点击"添加人物" → 输入姓名、描述、外貌、性格等
3. AI 续写时，系统自动将相关人物信息注入上下文中
```

### 6.3 润色已有内容

```
1. 在某个 Scene 中选中一段文字
2. 点击"AI 润色" → AI 优化文笔
3. 生成结果在 Draft 区显示 → 用户可对比原文
4. 确认后替换原文
```

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | AI 生成响应 < 30s（流式输出），页面加载 < 2s |
| 可靠性 | AI 调用失败时有重试机制（最多 3 次） |
| 数据安全 | 本地开发期间数据不离开用户机器；未来上云后启用 RLS |
| 可维护性 | TypeScript 严格模式，文件结构按功能模块组织 |
| 测试 | 核心业务逻辑有单元测试；AI 调用层有 mock |

---

## 8. 成功指标（MVP）

- 用户可以完成"创建故事 → 添加 Scene → AI 续写 → 审核确认"的完整流程
- 单次 AI 续写质量：生成内容与已有设定无明显矛盾
- 系统支持管理 100+ Scene 不出现明显性能问题

## 9. 术语表

| 术语 | 定义 |
|------|------|
| Canon | 正史——用户确认后的正式故事内容 |
| Draft | 草稿——AI 生成或作者初稿、用户尚未确认的内容 |
| Archived | 已归档——曾经是 Canon 但被废弃或替换的内容 |
| Alternative | 平行分支——非主线的替代叙事线内容 |
| Scene | 场景——故事的最小叙事单元 |
| Chapter | 章节——多个 Scene 的逻辑组合 |
| Branch | 分支——从某个节点分叉的平行叙事线 |
| Story Bible | 故事圣经——世界观、人物、地点等结构化设定集合 |
| Event Ledger | 事件账本——不可变的事件时间线记录 |
| Facts Added | Scene 级新增设定——该场景首次建立的世界观事实 |
| Open Threads | 未闭合叙事线——该场景中提出但尚未解决的伏笔/线索 |
| Prompt Context | 提示词上下文——每次 AI 调用时拼接的输入信息 |
| Source of Truth | 真相源——原文 (`scenes.content`) 是唯一真相源，摘要仅用于索引定位 |
